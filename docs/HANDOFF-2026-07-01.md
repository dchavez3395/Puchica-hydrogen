## 2. Environment constraints (critical — these WILL bite you)

- **Git is locked** in this authoring sandbox — cannot commit/merge/mv. The user
  (Daniel) commits on his own machine. Give him diffs/commands, don't try git.
- **The FUSE mount serves STALE bytes to shell tools.** `cat`/`node --check`/
  `esbuild`/babel via bash read outdated or truncated versions of files edited
  with the Edit/Write tools. TRUST THE FILE API (Read tool) and the user's
  `npm run build`. Do NOT trust shell reads of edited files. (esbuild in
  node_modules is the Windows binary — won't run in Linux either.)
- **Edits preserve CRLF**, so the user's git diff shows only real changes, not
  line-ending churn — good. Keep it that way (don't bulk-normalize).
- Working tree shows ~290 "modified" files that are pure CRLF↔LF noise — ignore.
