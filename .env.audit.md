## Risks identified

### 1. SHOPIFY_CLIENT_SECRET lives in plaintext in `.env`
- **Risk:** any process with read access to `.env` can mint Admin API tokens.
- **Mitigation:** `.env` is gitignored. Real exposure = local dev machine compromise.
- **Recommended:** keep `.env` permissions tight (chmod 600 on Unix). For deploy, Oxygen should inject this at deploy time, NOT have it baked into git.

### 2. SHOPIFY_CLIENT_SECRET is duplicated in `~/.openclaw/workspace/MEMORY.md`
- **Risk:** memory is plain markdown. If MEMORY is ever committed to a public repo, the secret leaks.
- **Recommended:** grep MEMORY for `SHOP_[REDACTED]_` and replace with `***`. Already done.
- **Note:** MEMORY is private, but defense-in-depth: don't store secrets in markdown.

### 3. SESSION_SECRET is in `.env`, not in `.env.example`
- **Risk:** low; `SESSION_SECRET` is auto-generated per project.
- **Action:** document it in `.env.example` (missing).

### 4. SHOPIFY_CLIENT_SECRET in `.env` differs from what `scripts/lib/shopify_admin.py` reads
- The script reads `SHOPIFY_ADMIN_CLIENT_SECRET` (with `_ADMIN_`).
- The `.env` has `SHOPIFY_CLIENT_SECRET` (no `_ADMIN_`).
- **Risk:** the script will fail with `Missing SHOPIFY_ADMIN_CLIENT_SECRET` unless you set it in the shell.
- **Recommended:** rename `.env` var to `SHOPIFY_ADMIN_CLIENT_SECRET`, OR rename the script's expected var to `SHOPIFY_CLIENT_SECRET`. Pick one. Suggest: rename `.env` var — script name is clearer.

### 5. No `.env.example` for the OAuth client_*
- **Risk:** new devs (or future-me in 6 months) won't know these vars exist.
- **Action:** add `SHOPIFY_CLIENT_ID=""` and a comment-only stub for `SHOPIFY_ADMIN_CLIENT_SECRET` (don't put a real value in the example).

### 6. `PRIVATE_STOREFRONT_API_TOKEN` is also `STORE_[REDACTED]_*`
- **Risk:** Medium. Token has read scope including customer accounts.
- **Action:** if not actively used, remove. Otherwise audit scope.

## Recommended edits

1. Rename `SHOPIFY_CLIENT_SECRET` -> `SHOPIFY_ADMIN_CLIENT_SECRET` in `.env` (and `.env.example` comment) so the script reads it without an extra shell step.
2. Add `SESSION_SECRET=""` to `.env.example` with a note: "auto-generated by `npx shopify hydrogen env pull`".
3. Add `SHOPIFY_CLIENT_ID=""` and `SHOPIFY_ADMIN_CLIENT_SECRET="<paste from dev.shopify.com>"` to `.env.example`.
4. Verify `PRIVATE_STOREFRONT_API_TOKEN` is used somewhere; if not, drop it.
